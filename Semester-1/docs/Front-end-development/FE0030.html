HTML table basics

Welcome to the lesson on Front-end web development. This would be a self-paced class that focuses on one of the fundamental topics in the HTML - Table. Please make sure that you are completing the entire lesson.

What are your learning goals in this lesson?​

GIF source: CLICK HERE

Goal 1: Learning what are HTML table basics.

Goal 2: Learning when should you NOT use HTML tables.

Goal 3: Learning different concepts in HTML tables.

Road map for learning this lesson

Milestone 1: Complete the “What is a table?” section. (the time required to read the “What is a table” section is 15 minutes).

What is a table?​

Image source: CLICK HERE

MDN describes “A table is a structured set of data made up of rows and columns (tabular data). A table allows you to quickly and easily look up values that indicate some kind of connection between different types of data”

Milestone 2: Complete the “Allowing cells to span multiple rows and columns” section. This section includes some interesting articles and videos.

Allowing cells to span multiple rows and columns​

MDN docs say that “ Sometimes we want cells to span multiple rows or columns. Fortunately, table headers and cells have the colspan and rowspan attributes, which allow us to do just those things. Both accept a unitless number value, which equals the number of rows or columns you want to be spanned. For example, colspan="2" makes a cell span two columns.

Click here to watch the video.(20 minutes)

Click here to practice multiple rows and columns (3 minutes)

Milestone 3: Quickly understand and learn the “Providing common styling to columns” section. (the time required to read this section is 10 minutes)

Providing common styling to columns​

Styling without <col>​

MDN shows that” HTML has a method of defining styling information for an entire column of data all in one place — the <col> and <colgroup> elements. These exist because it can be a bit annoying and inefficient having to specify styling on columns — you generally have to specify your styling information on every <td> or <th> in the column, or use a complex selector such as :nth-child.”

Styling with <col>​

According to MDN docs ” Instead of doing the above one, we can specify the information once, on a <col> element. <col> elements are specified inside a <colgroup> container just below the opening <table> tag.”

Click here to read and practice common styling with <col> and <colgroup> (3 minutes)
